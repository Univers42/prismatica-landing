# 🧭 Prismatica Onboarding: App Isolation & Protected Routing

### 🚪 The Secure Gate: `ProtectedRoute`
In `app/router/ProtectedRoute.tsx`, we've implemented our first "Security Controller". Unlike the landing page which is open to the public, the private dashboard requires a validated session.

- **Check State**: Before making a decision, the app enters a `CHECKING` state. This prevents that annoying "flash" where you see the login page for 100ms before it remembers you were logged in.
- **Redirect Logic**: We use React Router's `<Navigate replace />` so that if an unauthenticated user tries to "go back" to the dashboard after being kicked to login, the browser history doesn't create a loop.

### 🌓 Managing "Two Worlds" in one App
The big challenge of today was separating the **Immersive Landing** (full of stars, glows, and custom cursors) from the **Professional Workspace** (clean, static, high-productivity).

We solved this in the **`GlobalUI` orchestrator**:
1. It now consumes the `useLocation()` hook from the router.
2. We defined a list of `LANDING_ROUTES`.
3. If the user is on one of these routes, the ambient noise (`StarField`, `LightCursor`) is rendered.
4. If they move to `/dashboard`, these components are **lazy-unmounted**, freeing up GPU/CPU resources for actual data processing work.

### 🧬 The Core Logic: Supabase & Service Mapping
Today we transitioned from simple mock data to a real **Supabase Sandbox**.
- **FSD Neutrality**: Instead of having our UI depend directly on Supabase's `User` type, we created a `mapUser` function in our service layer. This is a critical FSD pattern; if we ever decide to move from Supabase to a custom Backend, we only change the mapper, not the 10+ components that use the user data.
- **Reactive Session Hub**: The `authStore` now listens to Supabase's internal auth state change. This is the heart of the app—if a session expires or is terminated via a separate tab, the whole app responds instantly.

### 📱 Responsive "Liquid" Sidebar
Finally, we realized that the Dashboard was unusable on mobile because the sidebar was hidden.
- **The Hamburger Morph**: We implemented a button in the Topbar that morphs from `Menu` to `X`.
- **Sliding Physics**: We used **Framer Motion** for the sidebar's entry. It doesn't just "appear"—it slides in with a `translateX` transition, providing a tactile, premium feel consistent with the landing page.
- **Backdrop Overlay**: We added a blurred backdrop. Not only does it look high-end, but it also serves as a giant "hit area" to close the menu. This is a best practice for thumb-reachability on mobile.
